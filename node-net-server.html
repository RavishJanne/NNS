Why distributed

The campus model is running out of land, power, and patience.

Hyperscale campuses still win on raw density. But they take years to permit, draw power in the hundreds of megawatts, and sit wherever land and utilities allow — rarely near the people generating the requests. NNS spreads the same silicon across the edge instead, where the power, the fiber, and the people already are.

Grid architecture

Node. Net. Server. The name is the architecture.

Three tiers, one continuous home-to-home mesh — from a spare closet to the regional core.

TIER 01 — HOME NODE

The home node

A closet-sized enclosure installed in a spare room, garage, or utility space. Runs our Ember silicon, contributes idle compute to the grid, and routes its waste heat back into the house.

1.2kW typical draw whisper-quiet heat-capture ready
TIER 02 — BLOCK HUB

The block hub

One hub per 40–120 homes. Aggregates traffic from nearby Home Nodes, caches hot model weights locally, and keeps the block running even if the backbone link drops.

sub-2ms local hop on-device caching automatic failover
TIER 03 — GRID CORE

The grid core

The regional exchange that stitches Block Hubs into the full NNS Grid. Handles orchestration, customer-facing APIs, and the long-haul links between regions.

multi-region routing enterprise API 99.98%+ uptime

The chip

Ember: silicon sized for a hallway closet, not a hall of racks.

Every Home Node and Block Hub runs the same die. We designed Ember around a different constraint than the rest of the industry — not how much compute a room can cool, but how much compute a household electrical panel and a homeowner's patience can tolerate.
